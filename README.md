# C++ Practice with Microcontrollers

A personal learning repository dedicated to exploring Embedded Systems, Modern C++, and low-level programming through hands-on projects.


## About This Repository

This repository serves as my personal laboratory for learning and experimenting with Embedded Systems development using C++. 
Instead of focusing solely on completing projects, this repository documents my journey toward understanding how software interacts with hardware at a much lower level than traditional application development.
This repository exists to bridge that gap.

# Objectives
The primary goals of this repository are:
- Learn modern C++ through practical embedded programming.
- Explore low-level programming concepts.
- Understand how microcontrollers communicate with hardware.
- Build clean and maintainable embedded software.
- Practice software engineering principles in resource-constrained environments.
- Develop a stronger understanding of computer systems beyond traditional application development.

# Why Microcontrollers?
I simply want to have other programming experiences besides web development.
I am also looking forward to learning about electronics and how I can integrate this learning into System development.

This repository is intentionally experimental.

Some projects may be simple.
Some code may later be rewritten.
Some implementations may not be optimal.

That is expected.

The purpose is not to produce perfect code immediately, but to document the learning process and continuously improve through iteration and experience.

Whenever possible, I prefer understanding *why* something works rather than simply making it work.
